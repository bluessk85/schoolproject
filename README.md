# 학교 일정 관리 협업 시스템

여러 사용자가 동시에 작업하며 업로드한 학교 일정 데이터를 처리하고, 이수 가능한 날짜를 계산하는 웹 애플리케이션입니다.

## 주요 기능

- **실시간 협업**: 여러 사용자가 동시에 접속하여 데이터 업로드 및 처리 과정을 공유
- **세션 공유**: URL을 통해 다른 사용자를 작업에 초대하여 동일한 작업 환경 공유
- **학교 데이터 처리**: 여러 학교의 일정 데이터를 업로드하고 통합 처리
- **날짜 추출**: 다양한 형식의 날짜 데이터 자동 인식 및 추출
- **이수 가능일 계산**: 업로드된 데이터를 기반으로 이수 가능한 날짜 계산 및 통계 제공
- **결과 다운로드**: 처리된 데이터를 엑셀 파일로 다운로드

## 설치 방법

1. 저장소 클론
```
git clone <repository-url>
cd 전학공
```

2. 가상환경 생성 및 활성화
```
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac
```

3. 의존성 설치
```
pip install -r requirements.txt
```

4. Firebase 설정
   - Firebase 계정 생성 및 프로젝트 생성: https://firebase.google.com
   - Realtime Database 및 Storage 설정
   - 프로젝트 설정에서 웹 앱 추가 및 API 키 확인
   - `.env.example` 파일을 `.env`로 복사하고 Firebase 설정 정보 입력

## 실행 방법

```
streamlit run app.py
```

웹 브라우저에서 자동으로 애플리케이션이 열립니다. (기본: http://localhost:8501)

## 협업 기능 사용법

1. **작업 세션 공유**:
   - 애플리케이션이 실행되면 사이드바에 공유 URL이 표시됩니다.
   - 이 URL을 다른 사용자와 공유하여 동일한 작업 세션에 초대할 수 있습니다.

2. **파일 업로드 및 공유**:
   - 사용자가 업로드한 파일은 자동으로 Firebase에 저장되어 다른 사용자와 공유됩니다.
   - 현재 세션에 업로드된 모든 파일 목록을 실시간으로 확인할 수 있습니다.

3. **실시간 처리 상태 공유**:
   - 한 사용자가 데이터 처리를 시작하면 다른 사용자에게도 처리 상태가 공유됩니다.
   - 모든 사용자가 처리 결과를 동시에 확인할 수 있습니다.

4. **데이터 초기화**:
   - "모든 데이터 초기화" 버튼을 클릭하면 세션의 모든 데이터가 초기화되어 새로운 작업을 시작할 수 있습니다.
   - 초기화는 모든 사용자에게 즉시 반영됩니다.

## 주의사항

- Firebase 무료 티어는 일일 트래픽 및 저장 용량에 제한이 있습니다.
- 실제 운영 환경에서는 적절한 보안 설정을 통해 데이터를 보호해야 합니다.
- 대용량 파일 처리 시 성능 저하가 발생할 수 있습니다.
